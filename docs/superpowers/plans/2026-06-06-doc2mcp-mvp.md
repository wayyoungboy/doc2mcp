# Doc2MCP MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and publish a v0.1 Doc2MCP CLI that compiles Markdown/text documentation into a portable MCP knowledge package with search, resources, prompts, citations, and a stdio MCP server.

**Architecture:** Doc2MCP is a package compiler, not a document converter. Importers normalize files into text sections, the compiler creates a reproducible package manifest and local index, retrieval uses deterministic lexical scoring for offline MVP behavior, and the MCP server exposes narrow tools/resources/prompts from the compiled package.

**Tech Stack:** Go 1.22+, standard library, JSON package format, stdio JSON-RPC for MCP, `go test ./...` verification, GitHub Actions CI.

---

## MVP Scope

- Build Markdown and plain text directories into `doc2mcp.json`, `index.json`, and copied source files.
- Preserve provenance: source path, section heading, byte offsets, checksum, package name, build version.
- Query compiled packages from CLI: `search` and `show`.
- Serve compiled packages through MCP tools: `search_docs`, `read_doc`, `cite_source`.
- Expose resources for compiled documents and prompts for source-grounded answers.
- Publish as a clean Go repository with README, docs, examples, CI, and release tag.

## Excluded From v0.1

- Native PDF/DOCX/PPTX parsing. Use MarkItDown or Docling upstream, then compile the Markdown output.
- Hosted package registry.
- Vector embeddings and seekdb adapter. The package format leaves room for them in v0.2.
- Remote URL crawling.

## File Structure

- `go.mod`: module metadata.
- `cmd/doc2mcp/main.go`: CLI entrypoint.
- `internal/model/model.go`: package, document, section, citation, search result types.
- `internal/importer/importer.go`: Markdown/text directory importer.
- `internal/compiler/compiler.go`: build package files.
- `internal/index/index.go`: lexical retrieval and section lookup.
- `internal/mcp/server.go`: stdio JSON-RPC MCP server.
- `internal/cli/cli.go`: command parsing and orchestration.
- `testdata/docs/`: sample docs.
- `README.md`, `docs/THREAT_MODEL.md`, `docs/RELEASE.md`, `.github/workflows/ci.yml`: release artifacts.

## Tasks

1. Write importer/compiler tests, then implement Markdown/text section extraction and package output.
2. Write index/search tests, then implement deterministic lexical scoring and citation IDs.
3. Write CLI tests, then implement `build`, `search`, `show`, `serve`, `version`.
4. Write MCP server tests, then implement `initialize`, `tools/list`, `tools/call`, `resources/list`, `resources/read`, `prompts/list`, and `prompts/get`.
5. Add release docs, examples, CI, license, and verify commands.
6. Commit, create GitHub repo `wayyoungboy/doc2mcp`, tag `v0.1.0`, and create release.
